# TensorScope 测试报告

## 测试环境
- Python 3.9.23
- TensorFlow 2.10.0
- PyTorch 2.8.0
- ONNX 1.14.0
- 环境：dltensorscope

## 测试结果

### 数学操作一致性
| 操作 | 状态 | 差异 |
|------|------|------|
| abs | 一致 | 无 |
| square | 一致 | 无 |
| sqrt | 不一致 | NaN |
| exp | 一致 | 无 |
| log | 不一致 | NaN |

### ONNX测试
- 模型创建：成功
- 模型验证：成功
- 推理执行：成功
- 结果一致性：与NumPy一致

### API映射
- ONNX到TensorFlow映射：164个操作
- 操作分类：正确识别伪操作和硬模板

## 问题分析

### 数值精度问题
- sqrt和log操作在负值输入时产生NaN
- 原因：不同框架对负数的处理策略不同
- 影响：需要特殊处理负数输入

### GPU库缺失
- CUDA相关库未安装（正常，CPU环境）
- 不影响CPU测试结果


## 结论

1. 大部分基础操作在不同框架间保持一致
2. ONNX模型能够正确执行
3. API映射完整准确
4. 主要问题集中在数值精度处理上
